<summary>
- Makes `teamvault-cli search` actually usable: each match now shows the secret NAME next to its key, not just an opaque key.
- Default output becomes an aligned two-column `KEY  NAME` table (name column blank when a secret has no name).
- Adds `--keys-only` to restore the old bare-key-per-line output for scripting.
- Adds `--limit N` to cap how many matches are returned/printed (0 = no limit).
- Follows TeamVault API pagination so results are no longer silently truncated to the first page.
- `--json` output changes shape: it now emits an array of objects `{key,name,username,url}` instead of an array of key strings (documented as a breaking output change).
- The search result now carries name/username/url directly from the search response â€” no extra per-key API calls.
- The hermetic fake server and the end-to-end scenario are updated to exercise and assert the new name-in-output behavior.
- README command reference and CHANGELOG are updated.
</summary>

<objective>

Names/usernames/urls come straight out of the existing search response â€” no new per-key requests.
</objective>
